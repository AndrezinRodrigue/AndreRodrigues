Web Responsive e CSS Elegant

Hoje não é segredo pra ninguém que boa parte dos usuários tem acesso por (laptops, tablets, netbooks, celulares, desktops com tela pequena, iMacs com telas gigantescas, segundo monitor etc.), ou seja, a internet já não tem possui barreiras em comunicação. Bom vamos ao foco do assunto sabemos que boa parte dos usuários hoje tem desde computadores tem em suas casas desde computadores de mesa até mesmo o celular e pensando nisso que vou falar sobre um assunto polêmico hoje na internet a (web-responsive). Ante começarmos a mostrar as técnicas de medidas adequadas é importante entendermos como surgiu a web responsive.

A Web responsive é uma técnica que surgiu com a ideia de solucionar o problema na questão acessibilidade, mas por onde começar a aplicar essa técnica? Devemos pensar em um layout que seja (acessível) a qualquer tela e que possa ser entendido por qualquer pessoa de forma simples, pois, não basta apenas ter containers que se ajustam a tela ainda precisamos nos focar no texto descritivo

Não confunda “Responsive Web Design” com “Adaptive Web Design”, pois, no primeiro caso conseguimos visualizar nosso layout em várias larguras em quanto no segundo só adapta em quatro modelos de telas.

Mobile First e Desk Last O Mobile first Nada mais que uma maneira de desenvolver primeiro lugar versão para dispositivos móveis e depois para telas de Desktops e Notebooks.

Desk last Em último caso Desktop que se compararmos ao Mobile First sua maneira de trabalho com os objetos é mais fácil

Média Querye

As Médias Queryes são usadas para definirem condições nas quais o css será usado, ou seja, se o dispositivo aceitar tais condições ele então exibirá x resultado algo do tipo isso:

@media(min-width: 768px){ header{ background-color: red; } }

Perceba que a partir de telas 768px teremos um background vermelho em nosso elemento. Medidas responsivas

O px é fácil de trabalhar em telas de 768px, mas para telas menores e até mesmo acima de 768px ele se torna chato de controlar objetos imagina só ter que fazer isto: @media(min-width: 300px){ header h1{ font-size: 3px; } } @media(min-width: 420px){ header h1{ font-size: 3px; } } @media(max-width: 768px){ header h1{ font-size: 10px; } } @media(max-width: 1200px){ header h1{ font-size: 10px; } }

Quando poderia apenas usar medidas responsivas que diminuiriam e muito seu código, mas quais medidas usar? Você adotará o uso de em e rem espaçamentos e tamanhos de fontes algo desse tipo:

header .container{ padding: 3rem 0; } @media(min-width: 768px){ header .container{ padding: 10rem 0; } }

header h1{ font-size: 3rem; } @media(min-width: 768px){ header h1{ font-size: 10rem; } }

Olha só que nosso código ficou mais clean se compararmos ao anterior que e as medidas que citamos até ajustam perfeitamente de acordo com o que queremos desde telas menores ou maiores.

Para largura e altura temos três opções %, vw e vh ambas se adaptam a telas menores de 320px e maiores de 768px veja alguns exemplos

header{width: 100%;}

Neste exemplo podemos ver que nosso objeto se ajusta em 100% da tela independente se for mobile ou Desk, mas também podemos usar medida da viewport sem nenhum problema veja como ficaria a largura

header{width: 100vw;}

No caso na altura os valores ficam se tornam fixos por tanto use em casos onde você irá ter uma altura definida para seu container veja como ficaria um exemplo: .container{ background-color: #333; height: 40vh; }

Caso queira que seu container ajuste o tamanho da altura automaticamente de acordo com conteúdo que irá ser inserido coloque da seguinte forma:

.container{height: auto;}

Dessa maneira ele se ajustara sem nenhum problema de acordo com o conteúdo inserido.
